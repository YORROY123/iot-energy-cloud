## 本機開發流程

### 推薦開發方式

不需每次都重建整個 compose stack。以下是常見場景：

**只改 backend 程式碼：**

```powershell
# 方法 A：直接重建 backend 容器
$env:DOCKER_BUILDKIT = "0"
docker compose up --build backend

# 方法 B：開發模式（掛載本地目錄，改程式碼自動重載）
# 在 docker-compose.yml 的 backend service 加上：
#   volumes:
#     - ./backend:/app
#   command: uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

**查看 log：**

```powershell
# 查看 backend log（即時）
docker logs iess-clone-backend-1 -f

# 查看所有服務 log
docker compose logs -f

# 查看特定服務最後 100 行
docker compose logs --tail=100 emqx
```

**進入容器 shell：**

```powershell
# 進入 backend 容器
docker exec -it iess-clone-backend-1 bash

# 進入 postgres 執行 SQL
docker exec -it iess-clone-postgres-1 psql -U iess -d iess_db
```

**查 InfluxDB 資料（Flux）：**

進入 `http://localhost:8086` → Data Explorer，貼上：

```flux
from(bucket: "realtime")
  |> range(start: -1h)
  |> filter(fn: (r) => r["_measurement"] == "sensor_data")
  |> filter(fn: (r) => r["device_uid"] == "fridge01")
  |> limit(n: 20)
```

## 新增設備類型

假設要新增 `door_sensor`（門磁感測器，value 為 0/1）。

### 步驟 1：更新模擬器

`simulator/main.py` — 在 DEVICES 清單中新增：

```python
DEVICES = [
    # ... 現有設備 ...
    ("C001", "S001", "door_sensor", "door01"),
]
```

模擬器的 payload 產生邏輯若需特殊值，調整 `simulate_value()` 函數：

```python
def simulate_value(device_type: str) -> float:
    if device_type == "door_sensor":
        return random.choice([0.0, 1.0])  # 0=關, 1=開
    # ... 其他類型 ...
```

### 步驟 2：更新前端看板

`dashboard.html` — 新增單位對應：

```javascript
const UNITS = {
  temperature: '°C',
  power: 'W',
  humidity: '%',
  co2: 'ppm',
  door_sensor: '',   // ← 新增
};
```

新增顏色樣式：

```css
.door_sensor .device-value { color: #fb923c; }
```

若需要客製化顯示（如 0/1 → 「關/開」），修改 `updateDevice()` 中的文字轉換：

```javascript
let displayValue = value.toFixed(1);
if (dtype === 'door_sensor') {
  displayValue = value === 1 ? '開' : '關';
}
document.getElementById(`val-${device_uid}`).textContent = displayValue;
```

### 步驟 3（可選）：新增告警規則

若需要門磁開超過 5 分鐘告警，在 PostgreSQL 手動插入規則：

```sql
INSERT INTO alert_rules (device_uid, condition, threshold, notify_email, is_active)
VALUES ('door01', 'gt', 0.5, 'admin@example.com', TRUE);
```

告警引擎（`mqtt_subscriber.py`）會自動套用現有的 `gt` / `lt` 判斷邏輯。

## 擴充告警規則

### 現有規則結構

| 欄位 | 說明 |
|---|---|
| `condition` | `gt`（大於）或 `lt`（小於）|
| `threshold` | 閾值（float）|
| `device_uid` | 套用此規則的設備 |

### 新增規則類型：`between`（範圍內為正常）

**Step 1** — 修改 `models.py` 的 AlertRule：

```python
class AlertRule(Base):
    # ...
    condition = Column(String, nullable=False)  # 'gt' | 'lt' | 'between'
    threshold_min = Column(Float, nullable=True)   # 新增
    threshold_max = Column(Float, nullable=True)   # 新增
```

**Step 2** — 修改 `mqtt_subscriber.py` 的判斷邏輯：

```python
for rule in rules:
    if rule.condition == "gt":
        triggered = value > rule.threshold
    elif rule.condition == "lt":
        triggered = value < rule.threshold
    elif rule.condition == "between":
        # 超出範圍才告警
        triggered = not (rule.threshold_min <= value <= rule.threshold_max)
```

**Step 3** — 執行 `init_db()`（或手動 ALTER TABLE）讓新欄位生效。

## 新增 REST API 端點

以新增「取得所有設備列表」為例。

**`backend/routers/data.py`** 中新增：

```python
@router.get("/devices")
def list_all_devices(db: Session = Depends(get_db)):
    devices = db.query(Device).all()
    return [DeviceSchema.model_validate(d) for d in devices]
```

確認 `DeviceSchema` 的 `model_config` 設有 `from_attributes = True`（`models.py` 已設定）。

**測試**：

```bash
curl http://localhost:8000/api/devices
```

新端點會自動出現在 `http://localhost:8000/docs`。

## 修改前端看板

`dashboard.html` 是純 HTML + 原生 JS，不需 npm 或 build step。直接修改後**重新整理瀏覽器**即可。

### 常見修改點

**改異常判斷閾值**（目前溫度 < -10°C 不算異常）：

```javascript
function isAnomaly(dtype, value) {
  if (dtype === 'temperature' && value > -15) return true;  // 改這裡
  if (dtype === 'power' && value > 2000) return true;       // 改這裡
  return false;
}
```

**增加溫度歷史折線圖**：

複製 `powerChart` 的 canvas 和 Chart.js 初始化程式碼，把 `POWER_DEVICES` 改為 `TEMP_DEVICES = ['fridge01', 'fridge02', ...]`，`updateChart()` 中加分支判斷 `device_type === 'temperature'`。

**切換為深色/淺色主題**：

CSS 變數化：在 `:root` 定義色彩變數，加 `.light-mode` class 覆寫，用 JS 切換 `document.body.classList.toggle('light-mode')`。

## 新增模擬器設備

`simulator/main.py` 的 DEVICES tuple 格式：

```python
("customer_id", "site_id", "device_type", "device_uid")
```

新增一台設備只需在 DEVICES list 加一行：

```python
("C003", "S005", "temperature", "fridge10"),
```

重啟 simulator 容器後，MQTT 會自動開始發送，backend 收到後會在 `devices` 表建立記錄（如果有對應的 `INSERT` 邏輯），或先只更新 InfluxDB。

> 目前 `mqtt_subscriber.py` 只 UPDATE 已存在的 `devices` 記錄，若要自動 UPSERT 新設備，改 UPDATE 為 `INSERT ... ON CONFLICT (device_uid) DO UPDATE ...`。

## 程式碼結構對照

```
backend/
├── main.py              ← 進入點。改啟動行為（middleware、lifespan）從這裡
├── ws_manager.py        ← 改推播邏輯（過濾特定客戶、壓縮訊息）從這裡
├── database.py          ← 改資料庫連線或加新的 DB 從這裡
├── models.py            ← 加新的 ORM model 或 Pydantic schema 從這裡
├── mqtt_subscriber.py   ← 改 MQTT 處理邏輯、告警邏輯從這裡
└── routers/
    ├── data.py          ← 加查詢 API 從這裡
    └── control.py       ← 加控制/排程 API 從這裡
```

## 常用指令速查

```powershell
# 啟動所有服務（前台，看 log）
$env:DOCKER_BUILDKIT = "0"; docker compose up --build

# 啟動所有服務（背景）
$env:DOCKER_BUILDKIT = "0"; docker compose up --build -d

# 停止並移除所有容器
docker compose down

# 停止並移除容器 + volume（清空資料）
docker compose down -v

# 查看運行中容器
docker compose ps

# 查看 backend log（即時）
docker compose logs -f backend

# 只重建並重啟 backend
$env:DOCKER_BUILDKIT = "0"; docker compose up --build -d backend

# 進 postgres 執行 SQL
docker exec -it iess-clone-postgres-1 psql -U iess -d iess_db

# 手動插入測試用告警規則
docker exec -it iess-clone-postgres-1 psql -U iess -d iess_db -c \
  "INSERT INTO alert_rules (device_uid, condition, threshold, is_active) VALUES ('fridge01', 'gt', -15, TRUE);"

# 查看 Redis 中的排程
docker exec -it iess-clone-redis-1 redis-cli KEYS "schedule:*"
docker exec -it iess-clone-redis-1 redis-cli GET "schedule:fridge01"
```
