## Windows + Rancher Desktop

### 安裝步驟

1. 下載並安裝 [Rancher Desktop 1.x](https://rancherdesktop.io/)
2. 啟動後，進入 **Preferences → Container Engine**，選擇 **`dockerd (moby)`**（非 containerd）
3. 等待狀態列顯示 `Running`

### 啟動平台

```powershell
# 取得程式碼
git clone https://github.com/YORROY123/yongyao-iot-platform.git
cd yongyao-iot-platform

# 關閉 BuildKit（避免 WSL2 segfault 問題，詳見疑難排解）
$env:DOCKER_BUILDKIT = "0"

# 建置並啟動所有服務
docker compose up --build
```

第一次執行會下載映像檔約 1.2 GB，需 3～10 分鐘。

### 確認全部就緒

```
emqx-1      | EMQX 5.x.x is running now!
postgres-1  | database system is ready to accept connections
backend-1   | Application startup complete.
backend-1   | MQTT connected — subscribing to iess/+/+/+/+
```

看到以上訊息代表全部正常。

## 環境變數說明

所有環境變數在 `docker-compose.yml` 中以 `environment:` 設定，也可建立 `.env` 檔覆寫：

| 變數 | 預設值 | 說明 |
|---|---|---|
| `POSTGRES_USER` | `iess` | PG 使用者名稱 |
| `POSTGRES_PASSWORD` | `iess123` | PG 密碼 |
| `POSTGRES_DB` | `iess_db` | PG 資料庫名稱 |
| `POSTGRES_URL` | `postgresql://iess:iess123@postgres/iess_db` | Backend 連線字串 |
| `INFLUX_URL` | `http://influxdb:8086` | InfluxDB 位址 |
| `INFLUX_TOKEN` | `my-super-secret-token` | InfluxDB API Token |
| `INFLUX_ORG` | `iess` | InfluxDB Organization |
| `INFLUX_BUCKET` | `realtime` | InfluxDB Bucket 名稱 |
| `MQTT_HOST` | `emqx` | MQTT Broker 主機 |
| `MQTT_PORT` | `1883` | MQTT Broker 埠 |
| `REDIS_URL` | `redis://redis:6379` | Redis 連線字串 |
| `DOCKER_INFLUX_USERNAME` | `admin` | InfluxDB 初始帳號 |
| `DOCKER_INFLUX_PASSWORD` | `admin123456` | InfluxDB 初始密碼 |

> **安全提醒**：上述預設值僅供本機測試。生產環境務必使用強密碼並透過 Secrets Manager 注入。

## docker-compose.yml 解析

```yaml
services:
  emqx:           # MQTT Broker
  influxdb:       # 時序資料庫（含初始化 bucket）
  postgres:       # 關聯式資料庫（含 healthcheck）
  redis:          # 快取 / 排程儲存
  backend:        # FastAPI（depends_on emqx + postgres healthy）
  simulator:      # 12 台虛擬感測器，3 秒一筆
```

### Healthcheck 設計

`backend` 使用 `depends_on` + `condition: service_healthy` 等待 EMQX 和 PostgreSQL 就緒，  
避免後端在資料庫未就緒前啟動導致連線失敗。

```yaml
backend:
  depends_on:
    emqx:
      condition: service_healthy
    postgres:
      condition: service_healthy
```

### 持久化 Volume

```yaml
volumes:
  influxdb_data:   # InfluxDB 資料不隨容器刪除
  postgres_data:   # PostgreSQL 資料不隨容器刪除
  emqx_data:       # EMQX session 持久化
```

## 雲端部署（VPS / AWS EC2）

### 最低 VPS 規格

- 2 vCPU / 4 GB RAM / 20 GB SSD
- Ubuntu 22.04 LTS

### 安裝 Docker

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
```

### 部署

```bash
git clone https://github.com/YORROY123/yongyao-iot-platform.git
cd yongyao-iot-platform

# 生產環境：使用 .env 覆寫密碼
cat > .env << EOF
POSTGRES_PASSWORD=your-strong-password
INFLUX_TOKEN=your-long-random-token
EOF

docker compose up -d
```

### 防火牆開埠

```bash
# MQTT（IoT Gateway 連入）
ufw allow 1883/tcp

# 管理後台（僅限內網，不要對外開放）
# ufw allow 18083/tcp  ← 不建議

# 應用後端 API（若前端在同機器可不開）
ufw allow 8000/tcp
```

## 疑難排解

### Q1: `DOCKER_BUILDKIT` Segmentation fault

**症狀**：`docker compose build` 輸出 `Segmentation fault (core dumped)` 並 exit 139

**原因**：Windows WSL2 + BuildKit 的已知 bug（BuildKit 使用 cgroup namespace 與 Rancher Desktop 衝突）

**解法**：
```powershell
$env:DOCKER_BUILDKIT = "0"
docker compose up --build
```

---

### Q2: `x509: negative serial number` / pull 映像失敗

**症狀**：`docker pull` 報 TLS 憑證錯誤

**原因**：Go 1.23+ 嚴格驗證 x509 憑證序號，部分 CDN 憑證不合格

**解法**：在 Rancher Desktop 的 WSL 內設定 registry mirror：

```bash
wsl -d rancher-desktop sh -c "printf '{\"registry-mirrors\":[\"https://mirror.gcr.io\"]}\n' > /etc/docker/daemon.json"
# 然後在 Rancher Desktop UI 重啟 Container Engine
```

---

### Q3: `docker-credential-wincred not found`

**症狀**：`docker compose` 報 credential helper 找不到

**原因**：`~/.docker/config.json` 中有 `"credsStore": "wincred"` 但 wincred 不在 PATH

**解法**：編輯 `C:\Users\{你}\\.docker\config.json`，移除 `"credsStore"` 那行：

```json
{
  "auths": {},
  "cliPluginsExtraDirs": ["C:\\Program Files\\Rancher Desktop\\resources\\resources\\win32\\docker-cli-plugins"]
}
```

---

### Q4: `relation "devices" does not exist`

**症狀**：backend log 報 PG table 不存在

**原因**：`main.py` 呼叫 `init_db()` 前沒有 import models，SQLAlchemy 的 `Base.metadata` 是空的，`create_all()` 不會建表

**解法**：確認 `main.py` 有：

```python
from database import init_db
import models  # ← 這行必須在 init_db() 之前
```

---

### Q5: `WritePrecision has no attribute 'SECONDS'`

**症狀**：InfluxDB 寫入報 AttributeError

**原因**：`influxdb-client` SDK 使用 `WritePrecision.S`（秒），不是 `SECONDS`

**解法**：`mqtt_subscriber.py` 中改為：

```python
.time(ts, WritePrecision.S)
```

---

### Q6: Backend 沒有正常起來

```bash
# 查看 backend 容器 log
docker logs iess-clone-backend-1 --tail=50

# 重建特定容器
docker compose up --build backend
```

### Q7: 看板顯示「重新連線中」

用 Chrome DevTools → Network → WS 頁籤確認連線狀態。
