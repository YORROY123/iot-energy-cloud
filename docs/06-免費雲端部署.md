# 06 — 免費雲端部署（Live Demo 指南）

> ⚠️ **本指南為 Render 舊版部署做法（仍可用，設定保留在 `render.yaml`）。**
> 目前線上 Demo 後端已改用 **Hugging Face Spaces + Neon（us-east）**，請參考
> **[13-HF版部署與架構改善.md](13-HF版部署與架構改善.md)**（含遷移原因與改善成效）。

架構：後端跑在 **Render.com**，資料庫用三個免費雲端服務，看板掛在 **GitHub Pages**。  
全部免費，不需綁信用卡（Render 本身也不需要）。

```
GitHub Pages                 Render.com
dashboard.html  ──WS──→  FastAPI backend (DEMO_MODE=true)
                              │         │         │
                         InfluxDB   PostgreSQL  Redis
                          Cloud      (Render)  (Upstash)
```

---

## 總覽：需要開的帳號

| 服務 | 用途 | 免費額度 | 需要信用卡 |
|---|---|---|---|
| [Render.com](https://render.com) | 後端 + PostgreSQL | Web 服務 750hr/月，PG 90天 | 否 |
| [InfluxDB Cloud](https://cloud2.influxdata.com) | 時序資料 | 30天保留期，5MB/次 | 否 |
| [Upstash](https://upstash.com) | Redis | 10,000 指令/天 | 否 |

---

## Step 1：開 InfluxDB Cloud 帳號

1. 前往 https://cloud2.influxdata.com/signup
2. 選擇 **Free Plan**（不需信用卡）
3. 選區域（隨意，建議 US East 或 EU Central）
4. 登入後，左側選 **Load Data → Buckets**
5. 點 **Create Bucket**，名稱填 `realtime`，Data Retention 填 `30 days`
6. 左側選 **Load Data → API Tokens**
7. 點 **Generate API Token → All Access Token**，複製 Token（只顯示一次！）
8. 記下以下三個值：
   - **URL**：頁面左上角網址列的 host，格式如 `https://us-east-1-1.aws.cloud2.influxdata.com`
   - **Token**：剛複製的
   - **Org**：左側選 **Settings → Organization**，複製 Organization Name

---

## Step 2：開 Upstash 帳號（免費 Redis）

1. 前往 https://upstash.com → **Sign Up**
2. 登入後，點 **Create Database**
3. 名稱隨意，Type 選 **Regional**，Region 選靠近的（如 `us-east-1`）
4. 點 **Create**
5. 建立後，在 Details 頁面找到 **Redis URL**，格式如：
   ```
   rediss://default:xxxxxxxxxx@xxx-xxx.upstash.io:6379
   ```
6. 複製這個 URL

---

## Step 3：部署後端到 Render

1. 前往 https://render.com → **Get Started for Free** → 用 GitHub 帳號登入
2. 點 **New → Blueprint**
3. 選擇你的 `iot-energy-cloud` repository
4. Render 會讀取 `render.yaml` 並自動建立：
   - 一個 **Web Service**（FastAPI 後端）
   - 一個 **PostgreSQL** 資料庫
5. 點 **Apply**，等候部署完成（約 3～5 分鐘）
6. 部署完成後，進入 **iot-energy-cloud** 這個 Web Service
7. 點左側 **Environment**，新增以下三個環境變數：

| Key | Value |
|---|---|
| `INFLUX_URL` | Step 1 的 URL |
| `INFLUX_TOKEN` | Step 1 的 Token |
| `INFLUX_ORG` | Step 1 的 Org Name |
| `REDIS_URL` | Step 2 的 Redis URL |

8. 設定完後，點 **Manual Deploy → Deploy latest commit**
9. 等 Deploy 完成，點 **Logs** 確認出現：
   ```
   Application startup complete.
   ```
10. 記下你的後端 URL，格式如：`https://iot-energy-cloud.onrender.com`

---

## Step 4：設定 GitHub Pages

1. 前往你的 GitHub repo → **Settings → Pages**
2. Source 選 **GitHub Actions**
3. 回到 repo → **Settings → Variables → Actions**
4. 點 **New repository variable**：
   - Name：`RENDER_BACKEND_URL`
   - Value：Step 3 的後端 URL（如 `https://iot-energy-cloud.onrender.com`）
5. 前往 **Actions** 頁籤 → 找到 **Deploy Dashboard to GitHub Pages**
6. 點 **Run workflow → Run workflow**（手動觸發一次）
7. 等 workflow 完成（約 1 分鐘）
8. 回到 **Settings → Pages**，會看到類似：
   ```
   Your site is live at https://YORROY123.github.io/iot-energy-cloud/
   ```

## 常見問題

### 看板顯示「重新連線中」

- 確認 Render 後端有正常 deploy（看 Logs 有無 error）
- 確認 `RENDER_BACKEND_URL` variable 設定正確（不要有結尾斜線 `/`）
- Render 免費版可能在冷啟動，等 30 秒後重整

### InfluxDB 寫入失敗

- 確認 `INFLUX_TOKEN` 正確（有區分大小寫）
- 確認 `INFLUX_BUCKET` 名稱是 `realtime`
- 在 Render Logs 搜尋 `InfluxDB write error`

### Render 的 PostgreSQL 90 天到期

到期後免費 DB 會被刪除，重新在 Render 建一個新的 PostgreSQL，更新環境變數即可。  
Demo 資料無需保留。
